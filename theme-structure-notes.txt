【WordPress テーマ構成メモ】

--------------------------------------------------
1. header.php
--------------------------------------------------
・サイト共通のヘッダー部分を出力するテンプレートファイル。
・役割：
  - <!DOCTYPE html> ～ <body> までの基本HTML構造を定義。
  - language_attributes() や bloginfo( 'charset' ) でWordPressの設定に合わせた情報を出力。
  - wp_head() を呼び出し、プラグインやテーマが必要なCSS・JSを挿入できるようにしている。
  - サイトタイトル（bloginfo( 'name' )）とキャッチフレーズ（bloginfo( 'description' )）を表示。
  - wp_nav_menu() を使って、primary というロケーションに登録されたメニュー（メインメニュー）を表示。

--------------------------------------------------
2. footer.php
--------------------------------------------------
・サイト共通のフッター部分を出力するテンプレートファイル。
・役割：
  - フッター内にコピーライト表記（年＋サイト名）を表示。
  - wp_footer() を呼び出し、プラグインやテーマがフッターに読み込むJSなどを挿入できるようにしている。
  - </body></html> でHTMLを閉じる。

--------------------------------------------------
3. functions.php
--------------------------------------------------
・テーマ全体の機能をまとめる設定用ファイル。
・役割：
  (1) テーマセットアップ（wp_practice_theme_setup()）
      - add_theme_support( 'title-tag' )
        → <title> タグをWordPress側に任せる設定。
      - add_theme_support( 'post-thumbnails' )
        → 投稿・固定ページでアイキャッチ画像（サムネイル）を使えるようにする。
      - register_nav_menus()
        → primary という場所に「メインメニュー」を登録し、header.php の wp_nav_menu() で使用。

  (2) ウィジェットエリア登録（wp_practice_widgets_init()）
      - register_sidebar() で primary-sidebar というIDのサイドバーを登録。
      - このサイドバーは sidebar.php で dynamic_sidebar( 'primary-sidebar' ) として読み込まれる。

・add_action() で after_setup_theme と widgets_init にフックし、
  WordPressの起動時に上記設定を反映させている。

--------------------------------------------------
4. sidebar.php
--------------------------------------------------
・サイドバー領域を出力するテンプレートファイル。
・役割：
  - is_active_sidebar( 'primary-sidebar' ) でウィジェットが設定されているかどうかを判定。
  - 設定されていれば dynamic_sidebar( 'primary-sidebar' ) で実際のウィジェットを表示。
  - 何も設定されていない場合は、「外観 → ウィジェット から追加してください」という案内文を表示。
・page.php から get_sidebar() を呼び出すことで、このサイドバーが表示される。

--------------------------------------------------
5. index.php
--------------------------------------------------
・投稿一覧ページ（ブログのトップなど）として動く、もっとも基本的なテンプレート。
・役割：
  - get_header() / get_footer() でヘッダー・フッターを読み込む。
  - have_posts() / the_post() のループで、投稿の一覧を表示。
  - 各投稿ごとに以下を出力：
      ・タイトル（the_title()）＋パーマリンク（the_permalink()）
      ・日付（get_the_date( 'Y.m.d' )）
      ・抜粋（the_excerpt()）
  - the_posts_pagination() によるページネーション（前へ／次へ）を表示。
  - 投稿が1件もない場合は「投稿が見つかりませんでした。」と表示。

--------------------------------------------------
6. front-page.php
--------------------------------------------------
・サイトの「フロントページ」（トップページ）用のテンプレート。
・役割：
  - get_header() / get_footer() でヘッダー・フッターを読み込む。
  - サイト独自デザインのメインコンテンツ（ホーム、店舗一覧、店舗詳細、お品書き、お知らせなど）を1ページにまとめて表示。
  - 「お品書き」セクションでは、SCF::get('menu_list') で取得したメニュー情報をカードレイアウトで表示。
  - 「お知らせ」セクションでは、投稿タイプ post から最新5件を取得し、日付とタイトルのリストとして表示。

・利用方法：
  - 管理画面 → 「設定 → 表示設定 → ホームページの表示」で
    「固定ページ」を選び、フロントページに指定した固定ページが存在するときに front-page.php が適用される。

--------------------------------------------------
7. page.php（固定ページ用）
--------------------------------------------------
・固定ページ（会社概要・アクセス・メニュー詳細など）を表示するテンプレート。
・役割：
  - get_header() / get_footer() でヘッダー・フッターを読み込む。
  - <main> 内を「コンテンツエリア＋サイドバー」の2カラム想定にしている。
  - ループ内で以下を表示：
      ・ページタイトル（the_title()）
      ・ページ本文（the_content()）
  - get_sidebar() を呼び出して、右側（または横）に sidebar.php の内容（メインサイドバー）を表示。
  - ページが見つからない場合は「ページが見つかりませんでした。」というメッセージ。

--------------------------------------------------
8. single.php（投稿詳細ページ用）
--------------------------------------------------
・個別の投稿（ブログ記事1本）を表示するテンプレート。
・役割：
  - get_header() / get_footer() でヘッダー・フッターを読み込む。
  - ループ内で以下を表示：
      ・記事タイトル（the_title()）
      ・日付（get_the_date( 'Y.m.d' )）
      ・本文（the_content()）
  - 下部に前後の記事ナビ：
      ・previous_post_link() で「＜ 前の記事」
      ・next_post_link() で「次の記事 ＞」
  - 記事が見つからない場合は「記事が見つかりませんでした。」というメッセージ。

--------------------------------------------------
9. 管理画面での設定ポイント
--------------------------------------------------
(1) メニュー設定
    ・「外観 → メニュー」でメニューを作成。
    ・表示位置で「メインメニュー（primary）」を選択すると、header.php のナビ部分に反映される。

(2) サイドバー（ウィジェット）設定
    ・「外観 → ウィジェット」で「メインサイドバー（primary-sidebar）」にウィジェットを追加。
    ・追加したウィジェットは page.php（固定ページ）などでサイドバーとして表示される。

(3) フロントページ設定
    ・「設定 → 表示設定 → ホームページの表示」で
        - 「固定ページ」を選択。
        - 「ホームページ」に任意の固定ページを指定。
    ・その状態で front-page.php がトップページとして使用される。
